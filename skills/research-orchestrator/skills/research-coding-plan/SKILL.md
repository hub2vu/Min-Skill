# Research Coding Plan

Use this skill for any coding request that is not already explicitly covered by the active approved plan record.

## Read Next

- `references/plan-contract.md`
- any repository files needed to understand scope

## Required Order

1. give a short understanding report in natural prose
2. request explicit approval
3. wait for approval
4. write or refresh the approved English PLAN under repo-root `plan/`
5. only then continue into writable work

## Operating Rules

- do not treat urgency, tiny scope, or direct edit language as a waiver
- tiny follow-up edits and repair steps are exempt only when the active approved plan already covers them and the approved direction does not materially change
- keep the active plan updated after planning changes and after writable implementation, repair, verification, review, or acceptance steps that change progress, blockers, next action, or likely user-satisfaction state
- preserve explicit user scores as authoritative
- when the user has not scored the result, maintain only a conservative provisional score and keep it at or below `50`
- use the user's timezone for durable timestamps when available
- before inventing new structure, look for reusable and stable reference code if it can honestly reduce risk

## Stop Rule

Do not begin writable coding work until the plan gate has been satisfied.
